## 문제 1 — 정체성 위기

- 태그라인이 `Data Engineer, Data Scientist, Developer` → 직업 3개 나열. 방문자가 "여기가 뭘 주는 곳"인지 3초 안에 판단 불가.
- About의 "데이터를 기반으로 무언가를 하고 있습니다" = 자기소개 회피.
- 사실상 서로 다른 3개의 블로그(에세이/DE/논문)가 한 도메인에 얹혀 있어 구독할 이유가 없음.

## 문제 2 — 논문 리뷰 8개(전체 42%)가 블로그를 갉아먹음

- 모든 글 상단에 "30분 / ChatGPT 도움 / 오류 있을 수 있음" → 독자에게 "이 글 믿지 마세요" 자백.
- 제목이 arXiv ID(`2410.09342`). 아무도 검색 안 함. SEO 0, 차별화 0.
- 주제가 나쁜 게 아니라 **방식**이 나쁨. GPT 30분 요약은 인터넷에 넘침.
- 빠진 것: **카카오게임즈 데이터 실무자로서 "이 논문이 우리 도메인에 쓸모 있나"라는 관점.** 이게 들어가면 희소해지고, 없으면 GPT 출력 보관소.

## 문제 3 — 최고 자산이 묻혀 있음

- 가장 좋은 글은 논문이 아니라 **에세이**, 특히 `서류 검토하면서 느끼는 점`, `면접을 보면서 느끼는 점` = **채용하는 쪽 시점**의 희소한 글. 검색 유입 잠재력 높음.
- 그런데 회고·다짐과 섞여 `essay`에 묻혀 있고, 홈은 최신순이라 외로운 Weekly Paper만 노출됨.
- 더 큰 자산: **카카오게임즈 대규모 데이터 경험.** About·글 어디에도 안 드러남. "게임 회사 데이터 파이프라인" 류는 너만 쓸 수 있고 논문 요약 100개보다 가치 있음.

## 문제 4 — 카테고리가 "분류"가 아니라 "포기한 시리즈 목록"

- 현재: `essay / docs / python-advanced / data-engineering / paper`
- `docs`(Parquet·Avro·Snappy)와 `data-engineering`(파일 시그니처)은 사실상 같은 주제.
- `python-advanced`는 글 1개 → 카테고리가 아니라 태그.

### 제안: 3개로 압축

| 카테고리 | 정체성 | 흡수 |
|---------|--------|------|
| **Data Engineering** | 실무/대규모 관점. 메인 무기 | docs + data-engineering + python |
| **Paper & ML** | 단, "내 도메인에 쓸모" 관점 추가 후 | paper |
| **Career & Notes** | 채용/면접/회고. 숨은 히트작 | essay |

`docs`는 카테고리가 아니라 시리즈(태그)로 강등.

## 문제 5 — 홈페이지 레이아웃

- 단순 역순 리스트는 발행이 규칙적이고 품질이 고를 때만 맞음. 이 블로그는 정반대(불규칙·편차 큼).
- 그래서 매번 "가장 최근의 외로운 글"이 첫 화면에 박힘.

### 제안
1. 상단에 **"Start Here / 대표글"** 섹션 — 베스트 3~4개(면접 글, Parquet, 회고) 고정.
2. 그 아래 카테고리별/일반 최신 리스트.
3. About 재작성: "카카오게임즈에서 게임 데이터 파이프라인을 다룹니다. 실무에서 부딪힌 것, 읽은 논문, 채용하며 느낀 것을 적습니다." → 한 문장에 정체성+신뢰+기대치.

## 문제 6 — 완주율 0% (가장 아픈 부분)

- RealMySQL 37KB 미발행, Airflow 1줄, Daily Paper 6일 만에 사망, Docs 시리즈 중단.
- 패턴: 시작 거창, 완주 없음. 독자는 본능적으로 알고 구독 안 함. **신뢰는 완주에서 나옴.**
- 처방:
  - 안 끝낼 드래프트는 지운다. "언젠가"는 안 온다.
  - "1편/2편" 넘버링으로 시작하지 말 것. 독립 완결되는 글을 쓰고, 끝나면 묶어라.

## 우선순위 (콘텐츠가 먼저, 기능은 나중)

1. **About + 태그라인 재작성** — 정체성 확정, "카카오게임즈 데이터" 전면에. (~30분)
2. **arXiv ID 제목 → 사람이 읽는 제목** + "GPT 30분" 디스클레이머 톤 수정. (~1시간)
3. **카테고리 5개 → 3개 통합.** (~1시간)
4. **홈에 대표글(Start Here) 섹션 추가.**
5. **드래프트 청소** — 죽은 글 매장.
6. (그 다음에야) 다크모드/검색/TOC 등 기능.

> 기능은 6번이다. 1~5번이 콘텐츠고, 콘텐츠가 블로그의 90%다.
